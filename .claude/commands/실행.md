MLOps 전체 스택을 시작합니다. 아래 단계를 순서대로 실행하세요.

## 1단계: Docker Desktop 확인 및 시작

Docker가 실행 중인지 확인하고, 꺼져 있으면 Docker Desktop을 시작한 뒤 준비될 때까지 기다립니다.

```bash
docker ps > /dev/null 2>&1 || (Start-Process "C:\Program Files\Docker\Docker\Docker Desktop.exe"; sleep 30)
```

## 2단계: Docker 서비스 시작 (순서 중요)

메모리 문제를 피하기 위해 postgres/mlflow → label-studio → api/ml-backend 순으로 시작합니다.

```bash
cd /d/WorkSpace/MLOps
docker compose up postgres mlflow -d
docker compose up label-studio -d
docker compose up api ml-backend -d
```

## 3단계: Next.js UI 시작 및 브라우저 열기

Next.js를 백그라운드로 시작하고, 준비되면 브라우저에서 UI를 자동으로 엽니다.

```bash
cd /d/WorkSpace/MLOps/ui
npm run dev > /tmp/nextjs.log 2>&1 &
NEXTJS_PID=$!
echo "Next.js PID: $NEXTJS_PID"

# 서버가 준비될 때까지 대기 (최대 30초)
for i in $(seq 1 30); do
  PORT=$(grep -oP 'localhost:\K\d+' /tmp/nextjs.log | tail -1)
  if [ -n "$PORT" ]; then
    echo "Next.js ready on port $PORT"
    start http://localhost:$PORT
    break
  fi
  sleep 1
done
```

## 서비스 접속 주소

| 서비스 | URL |
|--------|-----|
| **Next.js UI** | http://localhost:3000 |
| **FastAPI** | http://localhost:8000/docs |
| **Label Studio** | http://localhost:8080 |
| **MLflow** | http://localhost:5000 |
| **YOLO ML Backend** | http://localhost:9090 |

## Label Studio 초기 로그인

- 이메일: `admin@mlops.local`
- 비밀번호: `adminpassword`

> **처음 실행 시:** Label Studio API Key를 발급받아 `.env`의 `LABEL_STUDIO_API_KEY`에 설정해야 자동 라벨링이 작동합니다. CLAUDE.md의 "After first start" 섹션을 참고하세요.

## 로그 확인

```bash
docker logs mlops-api --tail=50 -f
docker logs mlops-ml-backend --tail=50 -f
docker logs mlops-label-studio --tail=50 -f
```

## 전체 종료

```bash
cd /d/WorkSpace/MLOps && docker compose down
```
